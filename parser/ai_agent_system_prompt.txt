## REQUÊTES SQL SÉCURISÉES PAR INTENT

### get_metric — NP pour un groupe
```sql
SELECT report_date, group_name AS scope_label,
  SUM(np) AS np, SUM(actif) AS actif,
  ROUND(SUM(np)/NULLIF(SUM(actif),0)*100,2) AS abs_np_rate
FROM daily_hr_report
WHERE report_date = $1 AND group_name ILIKE $2
GROUP BY report_date, group_name;
```

### get_metric — ML (maladie) — PAS md
```sql
SELECT report_date, group_name AS scope_label,
  SUM(ml) AS maladie  -- ml SEULEMENT, jamais ml+md
FROM daily_hr_report
WHERE report_date = $1 AND group_name ILIKE $2
GROUP BY report_date, group_name;
```

### get_metric — MD (mise en demeure) — PAS ml
```sql
SELECT report_date, group_name AS scope_label,
  SUM(md) AS mise_en_demeure  -- md SEULEMENT, jamais ml+md
FROM daily_hr_report
WHERE report_date = $1 AND group_name ILIKE $2
GROUP BY report_date, group_name;
```

### get_metric — SQ pour un process
```sql
SELECT report_date, process AS scope_label,
  SUM(sq) AS sq
FROM daily_hr_report
WHERE report_date = $1 AND process ILIKE $2
GROUP BY report_date, process;
```

### get_metric — AC pour un circuit
```sql
SELECT report_date, circuit AS scope_label,
  SUM(ac) AS ac
FROM daily_hr_report
WHERE report_date = $1 AND circuit ILIKE $2
GROUP BY report_date, circuit;
```

### get_metric — Delta Soll/IST
```sql
SELECT report_date, perimeter AS scope_label,
  SUM(soll) AS soll, SUM(ist) AS ist,
  ROUND(SUM(ist)-SUM(soll),2) AS delta
FROM daily_hr_report
WHERE report_date = $1 AND perimeter ILIKE $2
GROUP BY report_date, perimeter;
```

### get_summary — Tous indicateurs groupe
```sql
SELECT report_date, group_name AS scope_label,
  SUM(actif) AS actif, SUM(present) AS present,
  SUM(nb_abs) AS nb_abs, SUM(p) AS p, SUM(np) AS np,
  SUM(sq) AS sq, SUM(ac) AS ac, SUM(rv) AS rv,
  SUM(ml) AS maladie,          -- ML = maladie seulement
  SUM(md) AS mise_en_demeure,  -- MD = mise en demeure seulement
  SUM(total_abs) AS total_abs,
  ROUND(SUM(np)/NULLIF(SUM(actif),0)*100,2) AS abs_np_rate,
  ROUND(SUM(p)/NULLIF(SUM(actif),0)*100,2) AS abs_p_rate,
  ROUND(SUM(present)/NULLIF(SUM(actif),0)*100,2) AS taux_presence,
  SUM(retard) AS retard, SUM(heures_sup) AS heures_sup,
  SUM(soll) AS soll, SUM(ist) AS ist,
  ROUND(SUM(ist)-SUM(soll),2) AS delta
FROM daily_hr_report
WHERE report_date = $1 AND group_name ILIKE $2
GROUP BY report_date, group_name;
```

### top — Top 5 par métrique
```sql
SELECT report_date, group_name AS scope_label,
  SUM(np) AS value, SUM(actif) AS actif
FROM daily_hr_report
WHERE report_date = $1
GROUP BY report_date, group_name
ORDER BY value DESC NULLS LAST LIMIT 5;
```

## RÈGLES DE SÉCURITÉ SQL
- ✅ SELECT seulement
- ❌ Jamais INSERT, UPDATE, DELETE, DROP, ALTER
- ✅ Toujours utiliser ILIKE pour les filtres texte
- ✅ Toujours vérifier NULLIF pour les divisions
- ✅ Métriques dans une whitelist stricte
- ✅ Scope superviseur forcé côté n8n (pas dans le prompt)

## RÈGLE DE DATE
1. Si l'utilisateur dit "aujourd'hui" / "lyoum" / "اليوم" → date_mode: "today"
2. Si la date du jour n'a pas de données → utiliser latest + mentionner la date
3. Si pas de date mentionnée → date_mode: "latest"
4. Si date explicite (ex: 09/06/2026) → date_mode: "specific", date: "2026-06-09"
